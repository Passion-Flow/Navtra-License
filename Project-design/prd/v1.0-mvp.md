## 1. 产品定位

Forge 是**厂商专用的软件许可证签发与校验中枢**（License Authority）。所有在本根目录下开发的 B 端私有化产品都靠 License 收费；Forge 是唯一能签发 License 的系统，**只有厂商能用**，**绝不随产品交付给客户**。

- **Forge 是签发方**，自身**不被 License gate**（规则例外，`licensing.md` / b2b §11.1）。
- Forge 手握**所有客户产品的签发私钥** → 安全与备份要求按全局**最严**档执行（私钥丢失 = 全部客户 License 失效）。
- Forge 归类为**厂商内部产品**（b2b §11.1）：架构 / 多租户 / 白标 / 初始化向导可放宽；安全 / 备份 / 审计 / 认证 / 可观测 / 依赖 / API / DB / 缓存 / 任务 / 文件 / 性能**必须最严**。

## 2. 角色与边界（不可混淆）

| 角色 | 是谁 | 在哪运行 | 是否交付客户 |
|------|------|----------|--------------|
| **签发核心 `forge-api`** | 持 Ed25519 私钥 | **厂商内网 / 不进公网 ingress** | 否 |
| **公网校验边缘 `forge-edge`** | 无私钥，仅验签 + 记指纹 + 发租约 | 厂商**公网服务器**，重加固 | 否 |
| **管理前端 `forge-web`** | 厂商操作员 UI | 厂商内网 | 否 |
| **Verifier SDK** | 内嵌公钥 + 验签 + 采指纹 + 锁定态 | 嵌入**消费方产品** | 是（库，随产品） |
| 消费方产品 | 其他所有 B 端产品 | 客户私有化环境 | —— |

交付给客户的只有：① 产品（含内嵌 Verifier SDK + 厂商公钥）；② 一份远端签发的 License（在线短码 或 离线 `.forge`）。

## 5. License 双轨模型（核心）

| 维度 | 在线 License | 离线 License |
|------|--------------|--------------|
| 形态 | ~36 字符短码（UUID） | 自包含 `.forge`（payload + Ed25519 签名，base64 长串） |
| 校验 | 产品回连 `forge-edge`（phone-home） | Verifier 用内嵌公钥**本地验签**，断网可验 |
| 有效期载体 | 服务器 DB（短码仅查询键） | 写死 payload，签名保护 |
| 硬件绑定 | 首次激活产品**自动回传指纹**，服务器死绑首个硬件 | 客户**手工复制部署ID**给厂商签发时绑定 |
| 断网行为 | **租约 + 宽限期**（默认 24h 租约，超宽限转锁定） | 不依赖网络 |
| 防复制 | 服务器侧 **seat/实例追踪** + 硬件绑定，超额告警/吊销 | 硬件指纹不匹配即 fail-closed |
| 吊销 | 服务器侧**即时吊销** | **到期重签** + 可选**签名 CRL** 强吹销 |
| 适用 | 客户允许出网 | 客户完全隔离 / air-gapped |

**有效期档位**（签发时选）：`1个月 / 3个月 / 6个月 / 1年 / 3年 / 5年 / 永久`。永久 = 买断：**online perpetual = `active_until` 真 null**（服务器可即时吊销）；**offline perpetual = 签发日 + 99 年远期有限 `active_until`**（UI 显"永久(买断)"，保证加密信封始终有到期兜底，CRL 为主动召回）。1个月 = POC。

## 6. 功能清单（forge-web 管理端）

1. **仪表盘**：License 总数 / 即将到期 / 已吊销 / 各产品分布。
2. **产品**：完整 CRUD（slug / 名称 / features 模板 / quotas 模板 / 签名算法默认）。
3. **客户**：完整 CRUD（企业名 / 联系人 / 备注）。
4. **签发**：在线（填客户/产品/有效期档/订阅/quotas/features → 出短码）；离线（粘贴部署ID + 同上 → 出 `.forge`）。
5. **License 列表**：状态（issued/active/expiring/expired/revoked/locked）、绑定指纹（脱敏）、seat 用量、续期、替换、**吊销**、**删除**。
6. **审计日志**：append-only，可查所有内部操作，CSV 导出，仅 Super Admin / Auditor 可见。
7. **设置**：签名密钥管理（密钥列表 / 轮换 / 公钥导出，私钥永不显示）、Login Settings、Password Policy、2FA、邮件配置、CRL 生成。
8. **认证页**：登录 / 重置密码（厂商内部无自助注册）。

## 7. 验收标准（Given/When/Then）

**AC-1 加产品**
- Given 我已登录 forge-web，When 在「产品」页新增产品并填 slug/名称/features 模板/quotas 模板，Then 产品**完美落库**且签发页可选；When 我删除产品，Then 软删落库（`deleted_at`）+ 审计留痕，且不影响已签发 License 的历史记录。

**AC-2 在线签发 + 激活（硬绑定 + 租约+宽限 + 防复制）**
- Given 产品已建，When 我在「签发」选「在线」填客户/产品/有效期档/订阅/quotas/features，Then 生成 ~36 字符短码，落库 status=issued、seat 上限按签发设定、未绑定。
- When 客户产品激活页选「在线」粘贴短码，产品自动回传 `部署指纹 + cluster_id` 到 `forge-edge`，Then edge 校验短码有效且未超 seat → **首次绑定指纹** → 下发 `validation_token` + 24h 租约 → 产品解锁。
- When 客户断网，Then 租约 + 宽限期内照常运行；超宽限期才转锁定态。
- When 同一短码被复制到第 2 套硬件回连且超 seat，Then 拒绝 + 审计告警；我可一键服务器侧吊销，下次校验/续租即拒。

**AC-3 离线签发 + 激活（硬绑定 + 内嵌公钥本地验签）**
- Given 客户产品离线激活页自动生成唯一部署ID（硬件指纹），When 我在「签发」选「离线」粘贴该ID + 填客户/产品/有效期档等，Then 私钥签名生成自包含 `.forge`（payload + signature + metadata），base64 可复制。
- When 客户粘贴 `.forge` 进产品，Verifier SDK 用内嵌公钥验签 + 校验 `bound_fingerprint == 本机部署ID` + 有效期 + CRL，Then 全部通过才解锁，断网也能验。
- When License 被拷到别的部署 / 迁库换机，Then 指纹不匹配 → fail-closed 锁定。

**AC-4 吊销**
- 在线：When 我吊销某 License，Then 服务器侧即时生效，下次 activate/validate 拒绝（`LICENSE_REVOKED`）。
- 离线：When 我把某 License 加入 CRL 并生成签名 CRL，Then 客户产品导入 CRL 后该 License 即锁。

**AC-5 完整 CRUD + 删除（贯穿所有实体）**
- 所有实体（用户/客户/产品/License/密钥）增删改查**完美落库**；删除走软删 `deleted_at` + 审计；审计表本身 append-only **永不删**。

**AC-6 审计可查**
- Given 任一内部操作（登录/签发/激活/续租/吊销/替换/删除/改密/角色变更），Then 写入 append-only 审计表；When 我在「审计日志」页按时间/操作类型/资源筛选，Then 可查并导出 CSV。

**AC-7 消费方锁定态**（Verifier SDK 行为，非 Forge 自身）
- 无有效 License → 进程不退、后端拒一切受保护接口（`403 LICENSE_REQUIRED`）、前端只显 `需要激活许可证.` / `License activation required.`（逐字锁定）。

**AC-8 私钥不暴露**
- 任何路径（公网 edge / 日志 / API 响应 / 镜像 / 仓库）都拿不到签发私钥；edge 被打穿也无签发能力。

## 8. 非功能性需求

- **安全**：零破解风险；fail-closed；多点防绕过（非单一全局布尔）；时间防回拨；私钥 AES-256-GCM 静态加密 + KEK/DEK 分层；公网 edge 全 OWASP 加固 + WAF + 限流。
- **可用性**：在线校验 SLO 可用性 ≥ 99.9%（直接影响客户产品）；签发核心内网高可用。
- **性能**：edge 在线校验 P95 ≤ 200ms（热路径走 Redis）。
- **备份**：私钥**最严**备份（加密 + 异地 + 季度演练）；RPO ≤ 1h / RTO ≤ 4h。
- **审计**：append-only，保留 ≥ 1 年。
- **跨平台**：6 组合（Win/Linux/Mac × amd64/arm64）；镜像 multi-arch 双构建（注意 Oracle 无 arm64 镜像，需在文档标注限制）。

## 9. 显式不做（YAGNI 边界）

- 不做 Workspace 多租户（§11.1 厂商单租户）。
- 不做客户自助注册（厂商内部）。
- 不做白标 / Branding。
- 不做初始化向导 UI（厂商手动初始化 + CLI bootstrap）。
- Forge 自身不集成 License gating（签发方例外）。
